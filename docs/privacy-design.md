# 隐私与权限设计

- JWT 只从格式合法的 Bearer 头读取；无效令牌返回 401，不进入业务控制器。
- 普通用户只能访问自己的会话、消息、报告和角色卡状态。
- 用户画像、长期记忆、知识库、工具调试和全局风险数据仅管理员可见。
- 工具调用中的用户名由后端登录态注入，忽略客户端传入的身份字段。
- 短期上下文保存在 Redis，默认 24 小时过期；会话摘要、长期记忆和画像保存在 MySQL。
- 高风险事件单独落库，通过 Redis Stream 异步分发；Redis 不可用时回退到本地异步线程，避免阻断危机回复。
- 日志和接口响应不输出数据库密码、邮件授权码或 JWT 密钥；生产环境必须通过环境变量提供密钥。

当前系统属于心理支持工具，不替代医疗诊断。涉及真实用户时还应按部署地区要求补充数据保留期限、删除申请、审计日志和加密备份策略。
