# 隐私与安全边界

本文档只总结当前源码中已经能看到的隐私与安全边界，不把未来规划写成已实现能力。

## 已实现的安全基础

### 1. JWT 鉴权

后端通过：

- `JwtUtil`
- `JwtAuthenticationFilter`
- `SecurityConfig`

实现基于 JWT 的接口鉴权。

### 2. 接口权限分层

当前至少区分出：

- 公开接口
- 登录用户接口
- 管理员接口

### 3. 高风险消息处理

从 `MessageRouterService` 可见：

- 高风险消息会触发风险分析
- 高风险场景会尝试加锁避免重复告警
- 会记录高风险信息
- 会尝试发送邮件通知

### 4. 用户记忆与画像

系统会把聊天内容整理为：

- 会话消息
- 会话摘要
- 中长期记忆
- 用户画像

这意味着当前系统确实会长期保存敏感文本信息。

## 当前仍需谨慎的地方

### 1. 默认配置中存在敏感开发值

`application.yaml` 中包含：

- JWT secret 默认值
- 邮件配置
- 数据源默认连接信息

这些内容在真实部署时应迁移到环境变量或密钥管理系统。

### 2. 管理员接口读取能力较强

管理员接口可以查看：

- 所有用户画像
- 所有用户记忆
- 指定会话消息
- 全部报告

因此部署时必须严格控制管理员账号。

### 3. 测试接口不适合生产直接暴露

`/test/db` 和 `/test/mail` 更偏开发调试接口。

## 结论

从源码实际情况看，这个项目已经具备基本鉴权和权限分层，但仍属于开发态工程，距离严格的生产级隐私治理还有差距。
